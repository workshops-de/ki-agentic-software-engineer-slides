# Prompt Engineering

Clear communication is the difference between useful AI and frustrating AI.

---
layout: why
---

# Why Prompt Engineering Matters

Without structure, AI gives generic answers. With it, AI becomes a precision tool.

- A vague prompt returns a generic, often useless response
- A well-crafted prompt yields production-ready output on the first try
- Poor prompts waste time — you end up correcting more than creating
- Prompt engineering is a learnable, systematic skill — not intuition

---
layout: little-what
---

# What is Prompt Engineering?

The systematic design of inputs to AI models to achieve reliable, high-quality outputs.

# The Developer Productivity Gap

::left::

**Without Prompt Engineering**

- Vague requests → generic code
- No context → wrong assumptions
- One-shot thinking → brittle output
- More time correcting than building

::right::

**With Prompt Engineering**

- Specific requests → precise solutions
- Rich context → code that fits your codebase
- Iterative refinement → production-ready output
- Less correction, more building

# The Prompt Makes the Difference

::left::

**Vague**

```text
Write a Vue component
```

→ Generic, no requirements, no context

::right::

**Structured**

```text
You are a senior Vue.js architect.
Create a login form using Composition API
with TypeScript, Zod validation,
and Vitest unit tests.
```

→ Role, requirements, and defined output

::bottom::

The quality of AI output is a direct reflection of the quality of the input.
