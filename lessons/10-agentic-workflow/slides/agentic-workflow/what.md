# Agent vs. Tool

::left::

## Traditional Tool

- Requires explicit instructions
- Limited context understanding
- Single-purpose
- Human oversight needed

::right::

## Agentic System

- Interprets intent and context
- Multi-step problem solving
- End-to-end automation
- Human reviews final result

# GitHub Workflows: The Foundation

**GitHub Workflows** are automated processes that run on repository events. Core components:

- **Events**: push, pull_request, issues, schedule, workflow_dispatch
- **Jobs**: Sequential or parallel, conditional execution
- **Actions**: Reusable components from the marketplace
